# 🍸 Bar-Manager

**Bar-Manager** — это готовое open-source решение для создания электронного меню бара или ресторана. 

💡 **Идея проекта максимально проста:** 
Это "движок" для электронного меню. Любой желающий может взять этот проект, переписать клиентский фронтенд (`index.html`) под визуальный стиль и вайб своего заведения, и сразу начать им пользоваться! Вся скучная подкапотная магия — база данных, безопасность, админ-панель и API — уже написана, настроена и готова к работе. Вы просто меняете "витрину" и получаете кайф.

🤖 **Важное уточнение:**
Весь код Frontend-части (HTML, CSS и Vanilla JavaScript) в этом проекте был полностью сгенерирован ИИ. Он используется как рабочий Proof-of-Concept и базовый шаблон для интеграции с API бэкенда.

---

## 🛠 Технологический стек

Бэкенд проекта спроектирован по принципам **Чистой Архитектуры (Clean Architecture)** и **SOLID** для максимальной масштабируемости и независимости слоев.

*   **Язык:** Go (Golang 1.25)
*   **Роутер:** `gorilla/mux`
*   **База данных:** PostgreSQL 17 + `jackc/pgx/v5` (Connection Pool)
*   **Миграции:** `golang-migrate`
*   **Авторизация:** Custom JWT (JSON Web Tokens)
*   **Инфраструктура:** Docker, Docker Compose, Makefile
*   **Фронтенд:** Vanilla JS, HTML, CSS (без фреймворков)

---

## ✨ Основной функционал

1. **Клиентское меню:** Получение актуального списка коктейлей с ценами и составом в реальном времени.
2. **Панель Администратора:**
   * Безопасный вход по мастер-паролю (JWT).
   * Добавление новых позиций в меню.
   * Быстрое обновление цены (PATCH-запросы).
   * Удаление коктейлей из меню.
3. **Автоматизация:** База данных и миграции разворачиваются автоматически при старте контейнеров
---

## 🚀 Быстрый старт

Убедитесь, что у вас установлены **Docker** и **Make**.

1. **Клонируйте репозиторий:**
   ```bash
   git clone [https://github.com/N1ktarchik/Bar-Manager.git](https://github.com/N1ktarchik/Bar-Manager.git)
   cd Bar-Manager

```

2. **Настройте переменные окружения:**
Скопируйте пример конфига и установите свои значения (особенно `ADMIN_PASSWORD` и `SECRET_KEY`):
```bash
cp .env.example .env

```


3. **Запустите проект:**
Одной командой `make` мы собираем приложение, поднимаем базу данных и накатываем миграции:


```bash
make up

```


4. **Откройте приложение в браузере:**
* Клиентское меню: [http://localhost:8080/](http://localhost:8080/)

* Панель администратора: [http://localhost:8080/admin](http://localhost:8080/admin)




---

## 📜 Доступные Make-команды

Для удобства разработки предусмотрен `Makefile`:

* `make up` — Собрать и запустить все сервисы (app, db, migrations).


* `make kill-all` — Остановить контейнеры, удалить volume (БД) и очистить данные.


* `make restart-app` — Полный сброс и перезапуск проекта.


* `make logs` / `make app-logs` — Посмотреть логи контейнеров.

## 🌐 API Endpoints

Все эндпоинты работают через префикс `/api`.

**Публичные:**

* `GET /api/cocktails` — Получить меню.
* `POST /api/auth/login` — Авторизация админа, возвращает JWT.

**Защищенные (требуют заголовок `Authorization: Bearer <token>`):**

* `POST /api/cocktails` — Добавить новый коктейль.
* `PATCH /api/cocktails/{id}/price` — Изменить цену коктейля.
* `DELETE /api/cocktails/{id}` — Удалить коктейль.
